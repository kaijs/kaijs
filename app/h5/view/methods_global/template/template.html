全局空间是非常可怕的事情

存储空间

由于单页应用的复用，存储空间的私有性就非常重要。 同一个页面可能在多个应该场景中实例化，因此不同页面的存储空间必须独立，否则就会导致数据之间互相影像。 在单页应用中，对每一个页面初始化的时候生成独立的存储空间。

DOM节点

需要对ID节点进行添加后缀，以避免重复的ID存在。在某些第三方库中，往往默认为id是全局唯一的，如果在单页 应用中出现ID重复的节点，可能导致页面工作不正常。框架层在解析页面时可以将页面中的id全部添加后缀，以此 来保证全局页面中不存在任何重复的id。

CSS样式

在某个页面中定义的样式需要将其影像限定到特定的view空间 

                
                  [data-viewname="h5/view/methods_global"] {
                    color:'red'
                  }
如果存在复用相同页面需要使用相同样式的情况，可以使用redirect_viewname的命名空间 
                
                  [data-redirect_viewname="h5/view/methods_global"] {
                    color:'red'
                  }
